Critical minerals, energy, and a WTI dashboard

From deposit geography to intraday crude — building a deterministic data repo

Mar 17, 2026 · 04:00 UTC

Where it started

Brine and hard-rock deposits for battery-grade supply, public supply-chain data, and a parallel thread on energy commodities. The work split naturally into a versioned data story (static page below), a lightweight CL=F chart (Render), and a separate full WTI live build (flows + intraday chart, same repo).

Deposit geography

Supply is concentrated: the Andean brine triangle (Bolivia, Argentina, Chile), Australian spodumene (e.g. Greenbushes), US projects (Thacker Pass, Clayton Valley), Gulf and Midwest refining and export nodes, and gas benchmarks (Henry Hub, Permian hubs) plus LNG export capacity. China dominates much of downstream processing — that concentration shows up whenever you map trade and price series together.

Data architecture

Target layout for a reproducible public-data repo (ETL + validation + optional static notebook export):

critical-minerals-data/
├── processed/
│   ├── production_by_country.csv
│   ├── battery_metals.csv
│   ├── dispersed_metals.csv
│   └── energy/
│       ├── natural_gas.csv
│       ├── crude_oil.csv
│       └── lng_capacity.csv
├── etl/
│   ├── fetch_usgs_mcs.py
│   ├── fetch_eia_energy.py
│   └── validate.py
├── docs/
│   └── notebook.html
└── .github/workflows/etl.yml

USGS MCS and EIA are the main public anchors; Comtrade and other feeds are optional extensions.

Data story (static)

Vega-Lite panels in assets/critical-minerals-notebook.html — committed with the site. Swap for an Observable/Framework export later if you want the same path.

WTI — simple chart

Small Dash app in apps/wti-intraday/: candlesticks, volume, VWAP, stat bar, yfinance-backed. Good for a fast embed.

WTI — full live dashboard

Heavier app in apps/wti-live/ (from ~/wti-dashboard): FLOWS map + pipeline model and CHART tab, DuckDB, optional Databento and EIA keys via env. Second Render service — deploy from root render.yaml, then set iframe src to your live URL if the name differs.
